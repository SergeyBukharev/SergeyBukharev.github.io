Понимание задачи
Ситуация Проект — образовательная игра, в которой пользователи проходят сюжет, решая математические задачи.
По сценарию игроки перемещаются по станциям метро и для продвижения должны заходить на серверы, решая задания внутри интерфейса.
При этом у заказчика не было чёткого представления о том, как должен выглядеть интерфейс, и решение нужно было найти в процессе работы.
Задача Спроектировать интерфейс игры, который:
  • отражает игровой сценарий
  • поддерживает взаимодействие с заданиями
  • объединяет разные части интерфейса в единую систему
Проблемы Бизнеса:
  • Нет сформированного решения интерфейса
  • Необходимо разработать концепцию с нуля
  • Требуется найти формат, который поддерживает игровой процесс
Пользователей:
  • Неочевидно, как взаимодействовать с заданиями внутри игры
  • Необходимо связать обучение и игровой процесс
Аудитория
  • школьники
  • пользователи образовательных продуктов

Решение
Процесс работы Сначала были зафиксированы пользовательские сценарии, которые задавали основу взаимодействия. По ходу проекта они дополнялись и уточнялись.
Дальше шёл поиск решений:
  • визуализировались разные варианты интерфейса
  • обсуждались с заказчиком
  • дорабатывались и перерабатывались
Интерфейс формировался через последовательные итерации, а не по готовому ТЗ.
Интерфейс был построен вокруг игровых сценариев:
  • игрок находится в комнате на станции метро
  • взаимодействует с компьютером
  • решает задачи для продвижения
Ключевые решения:
  • интерфейс компьютера упростили, так как он дублировал элементы игрового экрана
  • проработали карту и переходы между локациями
  • интегрировали задания в сюжет через механику “математической капчи”
  • перебрали несколько вариантов стилистики и структуры
Макеты
Все баллы, награды и подсказки, которые игрок получает по ходу игры, складируются у него в инвентаре. Инвентарь
Небольшой онбординг рассказывает об основных элементах игры: наводишь на них мышь и получаешь необходимую информацию. Онбординг
Математические задачи интегрированы в сюжет так: для входа на сервер игроку нужно решить математическую капчу.
Решил — молодец, получи письмо от загадочного Алекса, в котором он расскажет, что нужно делать дальше.

Результаты
Проект показал, что интерфейс можно сформировать через итерационный процесс, даже при отсутствии чёткого ТЗ.
Фиксация сценариев и последовательный поиск решений позволяют прийти к рабочему результату.
